    Interface CompactionConfig

    Configuration for history compaction behavior.

    Compaction creates snapshot nodes in the Merkle-DAG that summarize the full CRDT state, allowing peers to skip replaying the full change history.

    gcAfterPrune

    gcAfterPrune: boolean

    After pruning the in-memory sync tree, delete the orphaned blocks from the Helia blockstore. Opt-in because deletion is destructive: once a block is gone, peers that lazy-load it via CollabswarmDocument.loadChangeBlock or that re-broadcast the change will not be able to fetch the data locally.

    Only blocks that are no longer reachable from the in-memory sync tree (and are not the snapshot boundary CID) are deleted, so accidentally re-attached ACL nodes are safe. CIDs remain in the in-memory _hashes set so duplicate incoming sync messages are still deduplicated.

    Has no effect when pruneAfterSnapshot is false.
