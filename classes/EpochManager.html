    • transitionEpoch(
          groupSecret: Uint8Array,
          members: Set<string>,
          reason: "member_added" | "member_removed" | "key_update",
          options?: { affectedMember?: string; algorithmName?: AesAlgorithmName },
      ): Promise<EpochTransition>

      Transition to a new epoch. Creates a child epoch whose parent is the current epoch and sets it as the new current epoch.

      Parameters

      • groupSecret: Uint8Array

        The new shared group secret from key agreement.

      • members: Set<string>

        The updated set of member public key hashes.

      • reason: "member_added" | "member_removed" | "key_update"

        The reason for the epoch transition.

      • Optionaloptions: { affectedMember?: string; algorithmName?: AesAlgorithmName }

        Optional settings for the transition.

        • OptionalaffectedMember?: string

          The public key hash of the added/removed member, if applicable.

        • OptionalalgorithmName?: AesAlgorithmName

          The AES algorithm for the encryption key (default: AES-GCM).

      Returns Promise<EpochTransition>

      The EpochTransition describing the new epoch and its cause.

      Remarks

      The 4th parameter changed from a positional affectedMember?: string to an options object in the AES-CTR/CBC support update so the algorithm choice cannot be silently swallowed by a stringly-typed positional arg. This is a breaking signature change, accepted because SwarmDB is in pre-1.0 alpha with no known live consumers; no overload shim is provided.
