      CRDTSyncMessage

      CRDTSyncMessage<ChangesType>: object
      • Defined in crdt-sync-message.ts:9

      CRDTSyncMessage is the message sent over both IPFS pubsub topics and in response to load document requests.

      tparam

      ChangesType A block of CRDT change(s).

      Type parameters

      • ChangesType

      Type declaration

      • Optional changeId?: string

        CID of the root change node.

      • Optional changes?: CRDTChangeNode<ChangesType>

        All document changes as an object who's keys are change object hashes and values are change objects or null. A null value means that the change should be fetched from an IPFS file (the IPFS filename is the hash).

        Data stored int the IPFS file is deserialized using a MessageSerializer implementation.

      • documentId: string

        ID of a collabswarm document.

      • Optional keychainChanges?: ChangesType

        Optional document keys list. Only populated while loading and receiving a document key update (due to the removal of an ACL reader).

        NOTE: Keychain changes should only ever be sent over encrypted libp2p streams (not IPFS pubsub).

      • Optional readersChangeId?: string

        CID of the root readers ACL change node.

      • Optional readersChanges?: CRDTChangeNode<ChangesType>

        An optional block of change(s) made to the reader ACL. undefined means no change was made to the reader ACL.

      • Optional signature?: string

        Signature of the sync message.

      • Optional writersChangeId?: string

        CID of the root readers ACL change node.

      • Optional writersChanges?: CRDTChangeNode<ChangesType>

        An optional block of change(s) made to the writer ACL. undefined means no change was made to the writer ACL.
